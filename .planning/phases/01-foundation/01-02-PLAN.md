# Plan 01-02: Create Layout Components

## Goal
Create the application shell with TopAppBar, SideNavBar, and MainWorkspace layout components matching the design system.

## Context
The layout follows "The Engineered Canvas" design philosophy from DESIGN.md. It's a dark, high-density technical interface with specific color tokens and typography. Reference the code.html prototype for exact visual implementation.

## Must Do

  - JSON.engine logo/title (Space Grotesk font)
  - Navigation links (Projects, Deploy, History)
  - Search bar with icon
  - Action buttons (cloud status, share, terminal)
  - User avatar
- [ ] Create SideNavBar component with:
  - Project info section (dataset icon, project name, filename)

- [ ] Create MainWorkspace component with:
  - Resizable split layout (60% canvas, 40% editor)

## Technical Details

### TopAppBar Specs
- Height: 48px (h-12)
- Background: #131313 (surface)
- Logo: Space Grotesk, bold, white
- Nav links: Inter, text-sm, tracking-tight
- Active link: white with sky-500 underline
- Inactive links: zinc-500, hover zinc-300
- Search: bg-surface-container-low, border-outline-variant/20

### SideNavBar Specs
- Width: 256px (w-64)
- Background: #0e0e0e (surface-container-lowest)
- Active item: bg-[#2a2a2a], border-l-2 border-sky-500, text-sky-400
- Inactive items: zinc-500, hover bg-zinc-800/30
- Font: Space Grotesk, uppercase, text-[10px], tracking-widest
- "New Node" button: gradient from-primary-container to-primary

### MainWorkspace Specs
- Flex layout with ml-64 (sidebar width)
- Left panel: 60% width (canvas area)
- Right panel: 40% width (editor area)
- Canvas area: bg-surface with canvas-grid class
- Editor area: bg-[#0e0e0e]

## Verification Steps
1. All three components render without errors
2. Layout matches code.html prototype visually
3. No horizontal scrolling
4. Responsive to window resize
5. Dark theme renders correctly
6. All Material Symbols icons display properly
