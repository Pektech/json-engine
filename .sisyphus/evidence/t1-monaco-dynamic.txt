# T1: Lazy-load Monaco Editor via next/dynamic

## Date: 2026-04-26

## Changes Made
- File: `src/components/editor/CodeEditor.tsx`
- Removed: `import * as monaco from 'monaco-editor'` (static import pulling ~3-4MB)
- Added: `import dynamic from 'next/dynamic'`
- Kept: `import type { editor } from 'monaco-editor'` (type-only, zero bundle cost)
- Kept: `import { useMonaco } from '@monaco-editor/react'` (just a React context hook)
- Replaced: `import Editor, { useMonaco } from '@monaco-editor/react'` → dynamic import
- Added: `EditorLoading` placeholder component
- Added: `const Editor = dynamic(() => import('@monaco-editor/react'), { ssr: false, loading: () => <EditorLoading /> })`

## Key Finding
The `import * as monaco from 'monaco-editor'` was shadowed by the `useMonaco()` hook return value.
All runtime `monaco` references (MarkerSeverity, editor.setModelMarkers, Range, languages.json)
come from the hook, NOT the static import. The static import was dead code.
